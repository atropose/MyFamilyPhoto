# MyPicManager 기획서

## 1. 프로젝트 개요

가족 구성원들의 핸드폰 및 SLR 카메라 사진·영상을 Linux PC에서 통합 관리하고,
가정 내외부 어디서든 웹 브라우저로 열람·검색·다운로드할 수 있는 **프라이빗 가족 미디어 서버**

---

## 2. 기능 요구사항

### 2.1 미디어 수집 및 처리

| 기능 | 설명 |
|------|------|
| **초기 전체 스캔** | 지정 루트 폴더 아래 전체 파일 탐색, EXIF 추출, 썸네일 생성 후 DB 저장 |
| **증분 업데이트** | DB에 없는 신규 파일만 탐색·처리 (수동 트리거 또는 스케줄) |
| **썸네일 생성** | 이미지: Pillow로 리사이즈 / 영상: FFmpeg로 대표 프레임 추출 |
| **EXIF 파싱** | 촬영 일시, GPS 좌표, 카메라 기종 등 추출 |
| **역지오코딩** | GPS 좌표 → 주소 변환 (Nominatim / OpenStreetMap, 무료) |

### 2.2 갤러리 및 검색

| 기능 | 설명 |
|------|------|
| **썸네일 그리드** | 촬영 날짜 역순 기본 정렬, 무한 스크롤 또는 페이지네이션 |
| **상세 보기** | 클릭 시 원본(또는 고해상도) 이미지·영상 전체화면, 이전/다음 탐색 |
| **날짜 검색** | 날짜 범위 선택으로 필터링 |
| **장소 검색** | GPS가 있는 사진만 대상, 저장된 주소 텍스트로 검색 (예: "제주", "서울") |
| **가족 구성원 필터** | 핸드폰 폴더명(가족 이름) 기반 필터 |
| **단일 파일 다운로드** | 사진·영상 원본 다운로드 |

### 2.3 영상 처리

- 브라우저 내 **인라인 재생** (HTML5 video 태그, MP4/MOV 기준)
- 재생과 동시에 **다운로드** 버튼 제공

### 2.4 주간 일기

| 기능 | 설명 |
|------|------|
| **일기 업로드** | 음성 파일 + 텍스트를 함께 업로드 |
| **주 기준** | 월요일 ~ 일요일 (ISO 주차) |
| **주간 뷰** | 업로드된 일기 + **해당 주에 촬영된 사진/영상** 함께 표시 |
| **일기 목록** | 주차별 목록 페이지 |

### 2.5 사진 필터링 (비가족 사진 제거)

**처리 흐름:**

```
스캔 실행 전 Windows AI 서비스 연결 확인
  ├─ 연결됨  → 스캔 + AI 얼굴 검출 병행
  └─ 미연결  → AI 없이 스캔 진행 (face_count = NULL 로 저장)
                → 나중에 Windows PC 켠 후 "AI 분석만 재실행" 가능

AI 분석 완료 후 → 관리자 검토 페이지 → 숨김/유지 결정
```

- **AI 검출 방식**: 얼굴 검출(InsightFace + CUDA) — 얼굴이 없는 사진(문서, 음식 등)을 후보로 분류
- **관리자 검토**: 후보 썸네일을 그리드로 표시, 개별 또는 일괄 숨김 처리
- **숨김 처리**: 파일 삭제 없이 DB의 `is_hidden` 플래그만 변경 (복구 가능)
- **수동 숨김**: 일반 갤러리에서도 우클릭/메뉴로 개별 숨김 가능
- **미분석 사진 재분석**: 관리자 페이지에서 Windows AI 서비스 연결 후 미분석 파일만 골라 AI 재실행 가능

### 2.6 인증 및 접근 제어

- 사이트 전체 단일 **패스코드** 로그인 (가족 공용)
- 세션 기반 인증, 브라우저 탭 닫아도 일정 기간 유지

## 3. 기술 스택

### Linux PC (상시 가동, 저전력)

| 영역 | 선택 | 이유 |
|------|------|------|
| **백엔드** | Python + FastAPI | 비동기 처리, AI 라이브러리 호환성 최적 |
| **데이터베이스** | SQLite | 파일 하나로 운영, 10만 건 이하 충분 |
| **프론트엔드** | Vue.js 3 + Tailwind CSS | 반응형 구현 용이 |
| **썸네일(이미지)** | Pillow | Python 표준, EXIF 처리 포함 |
| **썸네일(영상)** | FFmpeg | 영상 프레임 추출 및 스트리밍 |
| **역지오코딩** | Nominatim (geopy) | 완전 무료, OpenStreetMap 기반 |

### Windows PC (필요시 가동, RTX 4060)

| 영역 | 선택 | 이유 |
|------|------|------|
| **AI 추론 서비스** | Python + FastAPI | Linux 서버와 동일 언어, HTTP 통신 |
| **얼굴 검출 모델** | InsightFace + ONNX Runtime (CUDA) | RTX 4060 GPU 가속, 높은 정확도 |
| **실행 방식** | 수동 시작 (스캔 필요 시에만) | 전력 소비 최소화 |

## 4. 시스템 아키텍처

```
[Linux PC] ← 상시 가동 (저전력)
  ┌─────────────────────────────┐
  │  FastAPI 메인 서버           │  :8000
  │  SQLite DB (mypic.db)       │
  │  썸네일 폴더                 │
  └────────────┬────────────────┘
               │ HTTP (LAN, 내부망)
               │ 스캔 시에만 통신
               ↓
[Windows PC] ← 필요 시에만 가동 (RTX 4060)
  ┌─────────────────────────────┐
  │  AI 추론 서비스              │  :8100
  │  InsightFace + CUDA          │
  │  얼굴 검출 API               │
  └─────────────────────────────┘

원본 미디어 폴더 (Linux PC)
  ├── 홍길동/          ← 핸드폰 (가족 이름별)
  ├── 홍영희/
  ├── 홍철수/
  └── SLR/
        ├── 2024-03/   ← SLR 카메라 (날짜별)
        └── 2024-04/

외부 접근
  인터넷 → DDNS → 공유기 포트 포워딩 → Linux PC:8000
  (Windows AI 서비스는 외부에 노출하지 않음, LAN 전용)
```

## 5. 데이터베이스 설계

### `media_files` 테이블

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | INTEGER PK | |
| file_path | TEXT UNIQUE | 원본 파일 절대경로 |
| filename | TEXT | 파일명 |
| media_type | TEXT | `photo` / `video` |
| taken_at | DATETIME | 촬영 일시 (EXIF 우선, 없으면 파일 수정일) |
| week_year | INTEGER | 촬영 연도 |
| week_number | INTEGER | ISO 주차 번호 |
| gps_lat | REAL | GPS 위도 (없으면 NULL) |
| gps_lng | REAL | GPS 경도 (없으면 NULL) |
| address | TEXT | 역지오코딩 주소 |
| thumbnail_path | TEXT | 썸네일 파일 경로 |
| width | INTEGER | 원본 가로 (px) |
| height | INTEGER | 원본 세로 (px) |
| duration | REAL | 영상 재생 시간 (초) |
| family_member | TEXT | 폴더명 기반 가족 구성원 |
| is_hidden | BOOLEAN | 숨김 여부 (기본 false) |
| face_count | INTEGER | 검출된 얼굴 수 (NULL = 미분석) |
| ai_reviewed | BOOLEAN | 관리자 검토 완료 여부 |
| file_size | INTEGER | 파일 크기 (바이트) |
| created_at | DATETIME | DB 등록 일시 |

### `weekly_diaries` 테이블

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | INTEGER PK | |
| week_year | INTEGER | 연도 |

## 6. UI/UX 화면 구성

```
[로그인]
  패스코드 입력

[홈 / 갤러리]
  ├── 상단: 검색 바 (날짜 범위 / 장소 키워드 / 가족 구성원 필터)
  ├── 썸네일 그리드 (PC: 5열 / 태블릿: 3열 / 모바일 세로: 2열 / 모바일 가로: 3열)
  └── 썸네일 클릭 → [상세 보기]
        전체화면, 이전/다음 네비게이션, EXIF 정보, 다운로드, 숨김 처리

[주간 일기]
  ├── 주차 목록 (예: 2024년 15주차 · 2024-04-08 ~ 04-14)
  └── 주차 클릭 → [주간 뷰]
        ├── 음성 플레이어 (업로드된 경우)
        ├── 텍스트 일기 내용
        └── 해당 주 사진/영상 썸네일 그리드

[일기 업로드]
  음성 파일 선택 + 텍스트 입력 + 주차 선택 → 저장

[관리자 - 스캔]
  ├── Windows AI 서비스 연결 상태 표시 (연결됨 / 미연결)
  ├── [전체 스캔 실행] / [증분 스캔 실행] 버튼
  ├── [미분석 파일만 AI 재분석] 버튼 (AI 서비스 연결 시 활성화)
  └── 스캔 진행 상황 실시간 표시 (처리 중 파일명, 완료 수 / 전체 수)

[관리자 - 사진 검토]
  ├── 탭: 미검토 AI 후보 / 숨김 처리된 사진
  ├── 썸네일 그리드 (얼굴 없음 또는 face_count = 0 사진)
  └── 개별 또는 일괄 [숨김] / [유지] 버튼
```

**반응형 대응:**
- PC: 5열 그리드, 사이드 필터 패널
- 태블릿: 3열 그리드
- 모바일 세로: 2열 그리드, 햄버거 메뉴
- 모바일 가로: 3열 그리드

## 7. 스캔 처리 프로세스

```
[스캔 실행]
  │
  ├─ Windows AI 서비스 핑 확인
  │    ├─ 응답 있음 → AI 포함 스캔 모드
  │    └─ 응답 없음 → AI 없이 스캔 (face_count = NULL)
  │
  └─ 루트 폴더 탐색
       └─ 각 파일
            ├─ DB 존재 여부 확인 (file_path 기준) → 있으면 건너뜀
            ├─ EXIF 추출 (촬영 일시, GPS)
            ├─ 썸네일 생성 → 저장
            ├─ GPS 있으면 역지오코딩 → 주소 저장
            ├─ AI 서비스 연결 시 → 이미지 전송 → face_count 수신 저장
            └─ DB 저장

[AI 재분석 (별도 실행)]
  face_count = NULL 인 파일만 조회
  → Windows AI 서비스에 순차 전송
  → face_count 업데이트
```

## 8. Windows AI 서비스 API

Linux PC ↔ Windows PC 간 내부 HTTP 통신 전용 (외부 미노출)

| 엔드포인트 | 메서드 | 설명 |
|------------|--------|------|
| `/health` | GET | 서비스 상태 확인 (핑) |
| `/detect-faces` | POST | 이미지 바이너리 수신 → `{"face_count": 2}` 반환 |
| `/detect-faces/batch` | POST | 이미지 여러 장 일괄 처리 |

## 9. 배포 구성

```
공유기
  ├── DDNS 설정 (예: myfamily.duckdns.org)
  └── 포트 포워딩: 외부 443(또는 8000) → Linux PC 내부 IP:8000

Linux PC (상시 가동)
  ├── FastAPI 메인 서버 → systemd 서비스로 자동 시작
  └── (권장) Nginx 리버스 프록시 + Let's Encrypt SSL

Windows PC (필요 시 수동 가동)
  └── AI 추론 서비스 → 수동 실행 스크립트 (start_ai_service.bat)
```

## 10. 개발 마일스톤

| 단계 | 내용 | 우선순위 |
|------|------|----------|
| **1단계** | FastAPI 기본 구조, SQLite 연동, 파일 스캔 엔진, 썸네일 생성 | 핵심 |
| **2단계** | 갤러리 UI (Vue.js), 썸네일 그리드, 상세 보기, 패스코드 인증 | 핵심 |
| **3단계** | 날짜/장소/가족 검색, EXIF 역지오코딩 | 주요 |
| **4단계** | 영상 스트리밍·재생, 다운로드 | 주요 |
| **5단계** | 주간 일기 업로드·뷰 | 주요 |
| **6단계** | Windows AI 추론 서비스, 얼굴 검출 + 관리자 검토 페이지 | 부가 |
| **7단계** | 증분 스캔 스케줄러, 반응형 최적화, 배포 가이드 | 부가 |

## 11. 미결 결정 사항

| 항목 | 현황 |
|------|------|
| 영상 포맷 지원 | MOV(아이폰), MP4, AVI 등 — FFmpeg 트랜스코딩 여부 개발 시 결정 |
| HTTPS 적용 | Nginx + Let's Encrypt 권장하지만 선택 사항 |
| 스캔 자동 실행 | 수동 버튼 우선, 필요 시 cron 스케줄 추가 |
| AI 서비스 포트 | Windows PC 내부 IP:8100 (기본값, 설정에서 변경 가능) |
